# B站直播录制助手 — 设计文档

## 概述

面向主播和切片员的 B站直播自动录制工具。Web 面板管理主播列表，检测到开播自动录制，下播自动停止。v1 单主播监看，架构预留多主播扩展。

## 技术选型

- **后端**: Node.js (http 模块或 express)，单进程
- **录制**: ffmpeg，通过 child_process 管理子进程
- **前端**: 纯 HTML/CSS/JS，无框架，和现有记账 App 风格一致
- **状态检测**: 定时轮询 B站直播 API，30 秒间隔
- **数据存储**: JSON 文件（config.json），录制文件直写磁盘
- **通信**: 前端每 30 秒轮询 REST API

## 页面布局

仪表盘风格，分区卡片：

- **标题栏**: 应用标题 + 手动检查按钮 + 添加主播入口
- **监看中区**: 在线/直播中的主播卡片，显示状态、录制时长、停止按钮
- **离线区**: 已添加但离线的主播，可删除
- **录制文件区**: 最近录制列表，文件名、大小、时长，点击打开所在文件夹
- **设置区**: 保存路径自定义

## API 设计

为多主播预留数组结构：

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/status | 所有主播状态 + 录制列表 |
| POST | /api/streamer | 添加主播 {roomId} |
| DELETE | /api/streamer/:id | 删除主播 |
| GET | /api/settings | 获取设置 |
| PUT | /api/settings | 更新设置 {savePath} |
| POST | /api/check | 手动触发立即检查 |

## 录制流程

```
添加主播 → 离线
              ↓ 轮询/手动检测到开播
           直播中 → 自动启动 ffmpeg 录制
              ↓ 检测到下播
           离线 → 自动停止，文件保存
```

- 轮询间隔：30 秒
- 断线 ≤ 2 分钟：自动重连，续写同一文件
- 断线 > 2 分钟：关闭当前文件，重连后开新文件
- 手动停止：立即终止录制，不管主播是否还在播

## 文件存储

- 默认按主播名分文件夹：`./recordings/<主播名>/YYYY-MM-DD_HH-MM-SS.flv`
- 支持自定义保存路径（通过设置面板）
- 录制文件可直接用系统默认播放器打开
