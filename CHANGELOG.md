## 2026-05-14 — 画质选项、码率优化

### 按主播独立画质设置
- 每个主播卡片新增画质下拉选择器：原画 / 高清 / 标清 / 流畅
- 默认「原画」，切换即时生效并持久化到 config.json

### 流选择优化
- 反向遍历 B 站返回的 stream 数组，选最高画质流（原先取第一个=最低画质）
- 画质对应 qn 参数：原画=10000、高清=10000、标清=400、流畅=250

### MP4 编码优化
- 原画：CRF 18 + preset medium（视觉无损，文件较大）
- 高清：CRF 20 + preset fast
- 标清：CRF 23 + preset fast（默认档）
- 流畅：CRF 28 + preset ultrafast（最小文件）

> FLV/MKV/TS 使用 `-c copy` 无损复制源流，画质=源流画质，仅受选流 qn 影响

## 2026-05-14 — 主播归类、真实用户名、文件名优化

### 录像按主播归类
- 录像文件不再平铺在独立列表，直接嵌入每个主播卡片，可折叠展开（▶/▼ 切换）
- 删除主播时两步确认：先确认删除 → 再选是否连同视频文件一起删除
- 保留视频时文件进入「未归类录像」区块，可继续播放/转换/删除
- 未归类录像仅在有文件时才显示

### B 站真实用户名
- 修复用户名获取：从 `live_user/v1/Master/info` API 通过 UID 拿真实主播名
- 兜底逻辑：用户名 → 直播间标题 → 房间号
- 不再显示「房间XXXXXX」

### 文件名优化
- 新录制文件名格式：`主播名_时间戳.mp4`（如 `尽力局局长老二_2026-05-14T09-12-51.mp4`）
- 旧文件不改名，仅新录制生效

### 目录自动迁移
- 启动时扫描旧格式目录（`房间XXX` / 纯数字）并合并到当前主播名目录
- Poller 检测到名字变更时自动搬移录像
- 新增主播时也会检查并合并遗留目录

## 2026-05-14（早前）— 录制格式、FLV→MKV、防崩溃

### 多格式支持
- FLV（原始流 `-c copy`）/ MKV（推荐）/ TS（最稳定）/ MP4（重编码 libx264）
- 前端格式选择器，持久化到 config.json
- FLV 文件可一键无损转换为 MKV（`ffmpeg -c copy`）

### MP4 播放保护
- MP4 录制中 moov atom 未写入，无法播放
- 点击播放 MP4 时弹提示「录制中暂无法播放」

### 容错与稳定性
- ffmpeg 启动事件驱动检测（Windows 下 exitCode 不可靠）
- B 站 API 切换到 v2（`xlive/web-room/v2/index/getRoomPlayInfo`）
- 请求流时带 Referer + User-Agent 头防止 403
- 服务器重启时自动重置所有主播为离线状态
- ffmpeg 断连 2 分钟内自动重连复用同一文件，超过 2 分钟放弃

### 删除保护
- 删除正在录制中的文件时弹出「文件正在录制中，请先停止录制再删除」

## 2026-05-14（更早）— MVP 核心功能

### 后端 (server.js)
- Node.js 零依赖单进程服务（http、fs、path、child_process、https）
- 存储层：JSON 文件持久化（config.json）
- B 站 API 模块：`room/v1/Room/get_info` 查状态，`xlive/web-room/v2` 取流地址
- 录制模块：ffmpeg 子进程管理（启动/停止/状态查询）
- 轮询模块：30 秒间隔检测开播/下播，自动启停录制
- REST API：CRUD 主播、启停录制、查状态、查设置、改设置、打开文件、删除文件
- ffmpeg 自动发现：扫描 winget 安装目录

### 前端 (index.html)
- 编辑手记美学设计（暖色调、衬线标题、暗色模式支持）
- 添加主播：支持纯数字房间号 / live.bilibili.com 链接 / b23.tv 短链
- 监看中 / 离线 分区
- 手动检查按钮
- 录制文件列表
- 保存路径和输出格式设置
- 30 秒自动刷新

### 守护脚本
- start.bat / start-daemon.ps1：进程崩溃自动重启
