# 直播录制助手 (Bili Recorder)

自动监控 B 站直播间开播/下播状态，自动启动/停止 ffmpeg 录制。Web 面板管理，多主播支持，开箱即用。

## 系统要求

- **Windows** 10/11（macOS/Linux 理论上可用，未测试）
- **Node.js** ≥18 — [下载 LTS 版本](https://nodejs.org)
- **ffmpeg** — 安装方式：
  ```powershell
  winget install Gyan.FFmpeg
  ```
  或到 [ffmpeg.org](https://ffmpeg.org/download.html) 下载，将 `bin` 目录加入系统 PATH

## 快速开始（给朋友看这里）

```bash
# 1. 克隆项目
git clone https://github.com/rexthelinebarrel/bili-recorder.git
cd bili-recorder

# 2. 检查环境
双击 setup-check.bat

# 3. 启动
双击 start.bat

# 4. 打开浏览器
http://localhost:3456
```

## 使用说明

### 添加主播
输入框支持三种方式：
- **纯数字房间号**：如 `31001522`
- **直播间链接**：如 `https://live.bilibili.com/31001522`
- **b23.tv 短链**：如 `https://b23.tv/xxxxx`

### 主播卡片
每个主播有独立卡片，包含：
- 直播状态（直播中/离线）+ B 站跳转链接
- **画质选择**：原画 / 高清 / 标清 / 流畅（影响源流 qn 和 MP4 编码参数）
- **输出格式**：FLV / MKV / TS / MP4（按主播独立设置）
- 录像文件列表（折叠展开）
- 手动启停录制 / 删除主播

### 删除主播
点击「删除主播」时两步确认：
1. 确定删除主播？
2. 是否同时删除所有录制视频文件？
   - **确定** → 删除主播和所有视频
   - **取消** → 仅删除主播，视频出现在「未归类录像」区块

### 录制逻辑
- 每 30 秒轮询一次开播状态
- 开播 → 自动启动录制
- 下播 → 自动停止
- 断连 2 分钟内自动重连，复用同一文件
- 超过 2 分钟放弃，下次开播开新文件

### 后台运行
`start.bat` 启动后关闭 CMD 窗口即可停止。建议保持窗口最小化。

## 画质说明

| 档位 | 源流 qn | MP4 CRF | MP4 preset | 适用场景 |
|---|---|---|---|---|
| 原画 | 10000 | 18 | medium | 存档收藏，文件较大 |
| 高清 | 10000 | 20 | fast | 日常录制 |
| 标清 | 400 | 23 | fast | 节省空间 |
| 流畅 | 250 | 28 | ultrafast | 仅作记录，最小文件 |

FLV/MKV/TS 使用 `-c copy` 无损复制源流，画质取决于 B 站推流质量，不受编码参数影响。

## 文件存储

```
recordings/
├── 尽力局局长老二/
│   ├── 尽力局局长老二_2026-05-14T09-12-51.mp4
│   └── ...
└── 人形鹿头自走炮/
    └── 人形鹿头自走炮_2026-05-14T09-34-01.flv
```

保存路径可在「设置」中自定义。文件名格式：`主播名_时间戳.扩展名`。

## 功能一览

### 后端（server.js，零依赖）
- HTTP Server 内置 REST API
- B 站 v2 直播 API（流地址获取 + 用户名查询）
- ffmpeg 子进程管理（事件驱动，自动发现 winget 路径）
- 30 秒轮询检测开播/下播，自动启停
- 断连重连（2 分钟窗口，复用文件路径）
- JSON 文件持久化（config.json）
- 名字变更时自动迁移录像目录
- 启动时重置过期录制状态

### 前端（index.html，原生 HTML/CSS/JS）
- 编辑手记美学设计，暗色模式自适应
- 直播/离线分区，实时状态刷新
- 按主播归类的可折叠录像列表
- 一键播放（系统播放器）/ 删除 / FLV→MKV 转换
- 未归类录像处理
- 按主播独立的画质和格式设置

## 常见问题

**Q: 录像文件找不到？**
A: 默认保存在 `项目目录/recordings/<主播名>/`，可在面板「设置」中修改。

**Q: MP4 录制中无法播放？**
A: MP4 的 moov atom 在录制结束后才写入。录制中播放会弹出提示，停止录制后即可播放。或使用 MKV 格式（录制中也可播放）。

**Q: 视频画质差？**
A: 在主播卡片的画质下拉菜单中选「原画」。FLV/MKV/TS 是源流直存，画质 = B 站推流质量。MP4 格式选原画会使用 CRF 18。

**Q: 服务意外停止？**
A: 使用 `start.bat` 启动会自动重启。日志在 `app.log`。

**Q: 添加主播后显示「房间XXXXXX」？**
A: 等待 30 秒轮询后会自动更新为真实 B 站用户名。也可点击「立即检查」手动触发。
