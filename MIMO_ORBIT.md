# Bili Recorder — MiMo Orbit 百万亿 Token 计划申报案例

## 项目概述

**Bili Recorder（B站直播自动录制助手）** 是一个基于 Node.js 的 B站直播全自动录制工具。支持多主播同时监控、自动开始/停止录制、多格式多画质输出，运行在 Windows 环境下，CPU 占用 <1%，内存占用 ~30MB。

- **GitHub**: https://github.com/rexthelinebarrel/bili-recorder
- **技术栈**: Node.js + ffmpeg + B站 Live API v2 + 前端控制面板
- **当前状态**: 已持续运行录制，累计产出数十小时直播录像

---

## 现有能力

| 功能 | 说明 |
|------|------|
| 多主播监控 | 同时监控多个直播间，独立录制 |
| 自动启停 | Poller 轮询检测直播状态，自动开始/结束录制 |
| 多格式输出 | FLV / MKV / TS / MP4（MP4 支持 libx264 重编码） |
| 多画质选择 | 原画(10000) / 高清(10000) / 标清(400) / 流畅(250) |
| 直播流容错 | 断流自动重连，token 过期自动刷新 |
| 录制文件管理 | 按主播分目录，文件名含主播名+时间戳 |
| Web 控制面板 | 添加/删除主播、启停录制、格式切换、画质调节 |
| 格式转换 | 录制文件一键转封装（FLV→MKV 等） |

## MiMo 多模态能力集成路线图

### 阶段一：直播内容理解（使用 MiMo 多模态能力）

B站直播内容极其丰富——游戏、唱歌、聊天、教学、带货等。当前录制工具只能**全量录制**，用户需要事后手动寻找精彩片段。

MiMo 的多模态能力可以实时分析直播流，自动识别关键事件：

| 检测维度 | MiMo 能力 | 实现方式 |
|----------|-----------|----------|
| **弹幕密度峰值** | 文本理解 | 实时抓取弹幕流，MiMo 分析弹幕情绪、密度、关键词爆发 |
| **观众数突变** | 数值分析 | API 轮询观众数，MiMo 识别增长拐点 |
| **背景人声情绪** | 音频理解 | 直播流音频切片，MiMo 识别欢呼/尖叫/笑声/沉默 |
| **画面变化检测** | 视觉理解 | 关键帧抽取，MiMo 识别场景切换、特效爆发、击杀时刻 |
| **主播语音转文字** | 语音识别 | MiMo 实时 ASR，标记主播关键言论 |

### 阶段二：智能切片引擎

将以上多维信号融合，计算「精彩度评分」：

```
热度分数 = w1 × 弹幕密度 + w2 × 观众增速 + w3 × 人声能量 + w4 × 画面变化度
```

超过阈值时自动标记为高光时刻，自动裁剪为独立短视频片段。

### 阶段三：智能分发

切片自动生成标题、标签、封面（MiMo 视觉理解 + 文本生成），一键发布到短视频平台。

## MiMo Token 使用估算

| 环节 | 调用频率 | 单次 Token | 日 Token（8h直播） |
|------|----------|-----------|-------------------|
| 弹幕情绪分析 | 每 30s 一批弹幕 | ~2K tokens | ~1.9M |
| 观众数拐点检测 | 每 60s | ~500 tokens | ~0.24M |
| 音频人声分析 | 每 10s 音频切片 | ~5K tokens | ~14.4M |
| 画面关键帧分析 | 每 5s 一帧 | ~8K tokens | ~46M |
| ASR 语音转文字 | 持续 | ~10K/min | ~4.8M |
| **日均预估** | | | **~67M tokens** |
| **月均预估** | | | **~2B tokens** |

如果同时监控 3-5 个直播间，月消耗约 **6-10B tokens**，属于高价值、高消耗的应用场景。

## 为什么这个项目值得更多 Token

1. **真实用户需求**：B站有超过 100 万活跃主播，直播切片是内容创作者的核心需求，目前市场上缺乏高质量的开源自动切片工具
2. **多模态融合**：同时使用 MiMo 的文本、视觉、音频三大能力，是 MiMo 多模态能力的**标杆应用场景**
3. **持续消耗**：7×24 运行，Token 消耗稳定且持续，非一次性使用
4. **开源生态**：项目完全开源（MIT），可作为 MiMo 在直播内容理解领域的参考实现，吸引更多开发者使用 MiMo
5. **Agent 集成潜力**：未来可接入 Claude Code / Hermes Agent 等实现全自动运维（自动修 bug、自动适配 B站 API 变更）
6. **商业扩展性**：可发展为 SaaS 服务，服务 MCN 机构和独立主播
